R für empirische Wissenschaften

Kapitel 13 Reproduzierbare Berichte

In diesem Kapitel schauen wir uns an, wie man mithilfe von R Tabellen oder gleich ganze Berichte produzieren kann. Wieso sollte man das wollen, magst Du Dich fragen. Es kommt immer darauf an, was man macht. Für große Datensätze ist es praktisch, diese direkt in ein Textverarbeitungsprogramm einfügen zu können. Beim Absolvieren von Aufgaben ist es nützlich die Auswertung in ein Format zu bringen, welches auf jeden Computer ohne zusätzliche Programme angenehm gelesen werden kann (wie zum Beispiel HTML).

13.1 rMarkdown

Markdown ist eine so genannte Markup Language, also eine maschinenlesbare Sprache. Andere bekannte Markup Languages sind LaTeX, HTML oder XML. Im Vergleich zu den anderen genannten Vertretern ist Markdown extrem einfach zu bedienen. Die gesamte Formatierung geschieht im Hintergrund. Am häufigsten wird Markdown wohl verwendet, um es einfacher zu machen Inhalte für das Internet zu schreiben, welche üblicher Weise innerhalb von HTML geschrieben werden müssen, was recht unpraktisch ist. Der Nachteil sind fehlende Formatierungsoptionen, die LaTeX oder HTML zum Beispiel bieten.

Nun geht es in diesem Kapitel jedoch nicht um das klassische Markdown sondern um rMarkdown. rMarkdown ist nicht nur ein Package für R sondern auch ein eigener Datentyp, der mithilfe von R ausgeführt werden kann. Diese rMarkdown Dateien erlauben es uns zum Beispiel aus der intuitiv erstellbaren Markdown Sprache direkt HTML, PDF oder Word Dokumente zu erstellen, ohne irgendetwas am Inhalt zu ändern. Dabei sind einem beim Erstellen von HTML und PDF Dokumenten bezüglicher der Anwendung praktisch keine Grenzen gesetzt, während Word Dokumente aufgrund der proprietären Natur einige Limitationen aufweisen. Warum sollte man aber überhaupt ein HTML Dokument erstellen wollen? HTML (HyperText Markup Language) ist wenn man so möchte das Grundgerüst jeder Internetseite. Daraus resultiert, dass jeder Browser HTML Dokumente so übersetzen kann, als wäre es eine Website. Dafür benötigt man kein Internet, da Browser auch lokale HTML Dateien problemlos anzeigen können. Das hat den Vorteil, nicht auf zusätzliche Programme zum Öffnen der Dateien von anderen angewiesen zu sein, da jeder mindestens einen Browser auf seinem Computer hat. So können Ergebnisse schön angezeigt miteinander geteilt werden. In Addition mit dem dazu gehörigen rMarkdown File ist zusätzlich Reproduzierbarkeit gegeben, weil jeder mithilfe dieser Datei dieselbe HTML Datei erstellen kann. Auch im universitären Alltag kann dies von Nutzen sein um etwaige Hausaufgaben oder Assignments in gut lesbarer den Dozenten zukommen zu lassen.

Wir werden uns nun in Ruhe anschauen, wie so eine rMarkdown Datei denn eigentlich aussieht, wie man eine erstellt und wie man eine in andere Dateiformate umwandelt.

Abbildung 13.1: Typischer Aufbau eines R Markdown Skripts mit den wichtigsten Komponenten.

Was wir hier sehen, ist ein kurzes Beispiel für eine mögliche Hausaufgabe. Jedes rMarkdown Dokument (Endung .Rmd) beginnt mit einem so genannten YAML-Kopf, der durch drei Bindestriche oben und unten vom restlichen Dokument abgegrenzt ist. YAML steht für YAML Ain’t Markup Language. Dabei ist YAML im Prinzip nichts anderes als eine weiter vereinfachte Markup Language mit dem Hauptziel Optionen von Dateien festzulegen. In diesem Beispiel wird der Titel, Autor, das Datum und Outputformat festgelegt. Test muss dabei wie in R ebenfalls in Anführungszeichen geschrieben werden. Für mehrere Anpassungsoptionen sei auf das Buch R Markdown - The Definitive Guide von Xie, Allaire und Grolemund verwiesen.

Das nächste auffällige sind die grau hinterlegten Code Chunks. Das sind Abschnitte, die innerhalb der drei Backticks geschriebenen Code ausführen können und dessen Output direkt unterhalb innerhalb des Dokuments und nicht in der Console anzeigen. Nach den beginnenden drei Backticks folgen geschweifte Klammern. Die erste Information ist hier die verwendete Programmiersprache. Neben R können ebenfalls diverse weitere Sprachen wie C++ oder Python ausgeführt werden. Wenn wie beim ersten Code Chunk ein normales Wort (hier setup) auf die Programmiersprache folgt, ist das der Name des Code Chunks. Das ist bei größeren Projekten hilfreich, um den Überblick bei der Fehlersuche zu behalten, wo genau dieser aufgetreten ist. Grundsätzlich ist dies jedoch optional und nicht notwendig. Mit einem Komma getrennt können hier diverse Optionen spezifiziert werden. Häufig benötigt sind hier echo zum Festlegen der Sichtbarkeit des Codes und eval, um Code anzeigen zu können, der nicht ausgeführt werden soll. Für weitere Anpassungsmöglichkeiten sei auch an dieser Stelle auf das bereits genannte Buch verwiesen. Möchte man einen Code Chunks ausführen, kann man dies entweder wie gewohnt mit strg + enter oder durch Klicken auf den Grünen Pfeil in der oberen rechten Ecke.

Damit haben wir auch bereits fast alle Eigenheiten von rMarkdown gegenüber dem gewöhnlichen Markdown kennengelernt. Bevor wir zum Umwandeln in HTML, PDF und Word kommen, noch kurz ein paar Worte zu einigen Formatierungsmöglichkeiten innerhalb von Markdown. Eine Raute (#) steht für eine Überschrift. Je mehr Rauten geschrieben werden, desto kleiner wird die Überschrift, da diese Unterkapitel darstellen. Innerhalb von Asterisks geschriebener Text ist kursiv, bei doppelten Asterisks wird der Text fett. Die Backticks im Text (hier im ersten Abschnitt bei big_five) ermöglicht es, innerhalb des Textes Code zu schreiben. Dieser wird dann je nach Format grau hinterlegt hervorgehoben.

Nun bringt uns das Markdown Dokument relativ wenig, wenn es nicht in ein Dateiformat umgewandelt wird, was andere Menschen öffnen können. In der Leiste unter dem Reiter der geöffneten Dateien steht das Wort Knit (englisch für stricken) mit einem Symbol von Nähgarn. Wenn man nur auf das Symbol drückt, wird die Datei in das an erstellter Stelle im Output stehende Dateiformat umgewandelt. Besser ist es jedoch das Dropdown Menu durch Klick auf den Pfeil nach unten zu öffnen und den gewünschten Dateityp auszuwählen.

13.2 Tabellen exportieren

Während es selten ein Problem darstellt, die Ergebnisse einer einfach Regressionsanalyse tabellarisch im Textverarbeitungsprogramm des Vertrauens darzustellen, bereitet es schon deutlich weniger Freude eine große Korrelationsmatrix händisch von R zu übertragen. Glücklicherweise gibt es sowohl für Word als auch für LaTeX eine einfache Lösung, die automatisch eine Tabelle des gewünschten Datensatzes aus R erstellt.

13.2.1 Word und PDF mit rMarkdown

Es gibt verschiedene Möglichkeiten Tabellen im Word Format durch R Code zu erstellen. Wir werden uns hier auf das bereits in RStudio integrierte Pandoc und das in Kapitel 13.1 kennengelernte rMarkdown verlassen. Denn mit nur einer Funktion aus dem Package knitr, welches diverse Modifikationen für rMarkdown Dokumente bietet, können schöne Tabellen erstellt werden. Der Funktion kable() aus besagtem Package muss einfach nur der Datensatz übergeben werden. Dann muss man nur noch das rMarkdown Dokument mithilfe von Pandoc in ein Word Dokument umwandeln.

Das Aussehen muss dann innerhalb von Word entsprechend angepasst werden. Beachte die breite einer gewöhnlichen Din A4 Seite in Word. Wenn Du eine gigantische Korrelationsmatrix ausgeben möchtest, solltest Du entweder eine Word Vorlage in breitem Format beschreiben (die Vorlage muss den gleichen Dateinamen wie das rMarkdown Dokument haben), oder die Tabelle in Teilen ausgeben lassen.

13.2.2 LaTeX

Seit 1984 ist LaTex das Programm für wissenschaftliche Arbeiten und Bücher. Verwende MikTeX (https://miktex.org/) zur Verwendung auf Windows und LiveTeX (https://www.tug.org/texlive/) auf macOS und Linux. Wer eine sehr simple LaTeX Variante mit einfacher Installation haben möchte, kann sich einfach das Package tinytex installieren. Es gibt die Möglichkeit, tibbles aus R in LaTeX Code umzuwandeln, um diesen dann in die zugehörige TeX Datei zu kopieren. Dafür greift man auf die Funktion xtable() aus dem gleichnamigen Package zurück. Der Funktionen können einige Argumente übergeben werden, aber das wichtigste ist digits mit dem man pro Spalte die Anzahl der Nachkommastellen festlegt, auf die gerundet wird. Dabei kann man entweder eine einzelne Zahl eingeben und somit für alle Spalten die gleiche Rundung generieren oder für jede einzeln als Vektor mithilfe von c(). Da intern in der Funktion auch die Zeilennamen berücksichtigt werden, muss der Vektor immer um eins länger sein als die Anzahl der Spalten des eigentlichen Datensatzes.

big_five %>% 
  select(Extraversion, Neurotizismus) %>% 
  descriptive() %>% 
  xtable(digits = c(0, 1, 2, 2, 2, 2, 2, 2, 2, 2, 3)) %>% 
  print(include.rownames = FALSE)
