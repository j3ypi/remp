Kapitel 15 Namespace

Wenn man Packages lädt, kann es sein, dass diese Funktionen mit dem selben Namen verwenden (siehe Kapitel 7.1). Während beim tidyverse lediglich zwei selten verwendete base R Funktionen überschrieben werden, kann es beim Arbeiten mit vielen verschiedenen Packages, die nicht aufeinander abgestimmt sind, durchaus häufiger dazu kommen, dass mehrere Funktionen mit dem selben Namen geladen werden. Das ist, wenn man sich dessen nicht bewusst ist, eine Quelle für schwierig zu findende Befehle, weil sich die Funktion plötzlich nicht mehr so verhält wie erwartet. Dabei verwendet man automatisch die Funktion, die aus dem Package stammt, was als letztes geladen wird. Lädt man beispielsweise zuerst das Package tidyverse und anschließend data.table, erhält man folgende Meldung,

library(tidyverse)
library(data.table)
Attaching package: ‘data.table’

The following objects are masked from ‘package:dplyr’:

    between, first, last

The following object is masked from ‘package:purrr’:

    transpose

welche uns mitteilt, dass drei Funktionen (between(), first(), last()) aus dem Package dplyr und eine Funktionen (transpose()) aus dem Package purrr überschrieben werden. Es ist also wichtig, diese Meldungen beim Laden eines Packages durchzulesen. Glücklicherweise gibt es auch hier einen Weg, dies zu umgehen. Man kann Doppelpunkte mit der Syntax package::funktion() verwenden, um zu spezifizieren, aus welchem Package die Funktion kommt. Um die trotz des späteren Ladens von data.table auf die between() Funktion von dplyr zuzugreifen, würde man also beim Aufrufen der Funktion folgendes schreiben:

dplyr::between(1:12, 7, 9)

Da das tidyverse nur andere Packages lädt kann man nicht tidyverse::funktion() schreiben, sondern muss das Package aus dem die Funktion stammt (z.B.dplyr) direkt ansprechen.

Für genauere Informationen zum Thema Namespace und die Hintergründe der damit verbundenen Environments sei auf Advanced R von Hadley Wickham verwiesen.
